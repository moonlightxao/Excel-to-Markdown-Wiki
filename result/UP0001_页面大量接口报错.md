# PL-UP0001-页面大量接口报错

## 故障描述

用户登录前端页面发现大量接口报错 500。

## 定界手段

### 定界手段 A0001-排查关键依赖平台是否异常

**定界工具**：监控平台

**定界结果**：关键依赖异常

**定界步骤**：
1. 通过关键平台接口拨测，检查关键依赖是否出现异常
2. 通过日志监控，检查当前是否有关键依赖报错的日志

### 定界手段 A0002-排查数据库 CPU 使用率是否异常

**定界工具**：数据库监控平台

**定界结果**：数据库 CPU 使用率异常

**定界步骤**：
1. 检查数据库 CPU 使用率指标，若连续十个周期达到 95% 以上，说明 CPU 满导致的接口无法正常响应

## 恢复方案

### 恢复方案 R0001-联系关键依赖平台支持人员协助处理

**恢复工具**：不涉及

**恢复步骤**：
1. 根据沟通矩阵联系关键依赖平台人员支持处理

### 恢复方案 QR0002-数据库 CPU 异常恢复方案

**恢复工具**：数据库管理平台

**恢复步骤**：
1. 检查当前是否有大量慢 SQL 占用 CPU，若有大量慢 SQL 则开启自动查杀会话
2. 若执行 1 数据库的 CPU 仍未下降，考虑进行主备切换

> [!WARNING]
> 执行主备切换属于高风险操作，可能导致短暂的服务中断，请确保已通知所有相关干系人并准备好回滚方案。
